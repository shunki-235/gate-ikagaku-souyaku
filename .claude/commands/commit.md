git status、git diff（staged + unstaged）、git log --oneline -5 を並列で実行し、変更内容を把握する。

以下のルールに従ってコミットを作成する。

## コミットメッセージ形式

```
<prefix>: <何を実装/修正したかの具体的な説明>
```

### prefix一覧

| prefix | 用途 |
|--------|------|
| feat | 新機能の追加 |
| fix | バグ修正 |
| refactor | リファクタリング（挙動は変えない） |
| chore | 雑務・設定変更（ビルド/依存/運用など） |
| docs | ドキュメントのみの変更 |
| style | 見た目/フォーマット調整（ロジック変更なし） |
| test | テスト追加・修正 |

### 例

- `feat: FAQページのアコーディオン機能を追加`
- `fix: ヘッダーナビのリンク切れ`
- `refactor: お問い合わせフォームのバリデーション周辺を整理`
- `chore: pnpm + browser-sync の開発環境設定`

## ルール

- Co-Authored-By は付けない
- 変更に関連するファイルのみステージする（git add で個別指定）
- 無関係な変更は含めない
- メッセージは簡潔に、何を実装/修正したか明確にする
